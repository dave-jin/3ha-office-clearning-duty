## 1. 프로젝트 개요

### 1-1. 문제 정의

현재 사무실 청소 당번제(v1.1)를 운영 중이나, 다음 문제들이 존재한다.

| 문제                            | 영향                                  |
| ------------------------------- | ------------------------------------- |
| 당번 정보가 즉시 파악되지 않음  | 매주 "이번 주 누구지?" 반복 질문 발생 |
| 구성원 변동 시 수동 재편성 필요 | 입/퇴사마다 관리자 수작업, 편성 지연  |
| 대체자 변경 내역이 흩어져 있음  | 누가 바꿨는지 추적 어려움             |
| 체크리스트 완료 확인 불가       | 청소가 제대로 됐는지 알 수 없음       |
| 공지가 수동 발송                | 관리자가 까먹으면 알림 자체가 안 감   |

### 1-2. 솔루션 요약

**웹 기반 청소 당번 보드**를 구축한다.

- CEO 포함 **전원 참여**, 예외 없는 당번제
- **2인 1조 고정** — 홀수 인원 시 1명 프리패스 (3인조 없음)
- 구성원 추가/제거 시 **자동 라운드 재편성**
- 매주 목요일 청소 전 **Slack 자동 알림** 발송
- 대체자 변경, 스킵, 체크리스트까지 한 곳에서 관리

### 1-3. 성공 지표

| 지표                            | 목표                     |
| ------------------------------- | ------------------------ |
| 당번 확인까지 걸리는 시간       | 보드 접속 후 3초 이내    |
| Slack 알림 자동 발송률          | 100% (수동 개입 0)       |
| 구성원 변동 후 재편성 소요 시간 | 관리자 조작 후 즉시 반영 |
| 체크리스트 완료 기록률          | 90% 이상                 |

### 3-2. F1: 2인조 고정 + 프리패스 시스템

이 PRD의 핵심 설계. **어떤 인원이든 항상 2인 1조만 편성**하고, 홀수 인원일 때는 1명에게 프리패스를 부여한다.

#### 3-2-1. 기본 공식

```
활성 멤버 수 = N

짝수 (N = 2k):
  → k조 (2인×k), 프리패스 없음
  → 라운드 = k주

홀수 (N = 2k+1):
  → k조 (2인×k) + 1명 프리패스
  → 라운드 = k주
  → 프리패스 멤버는 다음 라운드 Week 1에 우선 배정
```

#### 3-2-2. 인원별 라운드 구조

| 활성 멤버 | 조 수 | 라운드 주차 | 프리패스   | 비고          |
| --------- | ----- | ----------- | ---------- | ------------- |
| 15명      | 7조   | 7주         | 1명/라운드 | **현재 상태** |
| 14명      | 7조   | 7주         | 없음       | 완전 페어     |
| 16명      | 8조   | 8주         | 없음       | 완전 페어     |
| 13명      | 6조   | 6주         | 1명/라운드 | 프리패스 발생 |
| 12명      | 6조   | 6주         | 없음       | 완전 페어     |
| 11명      | 5조   | 5주         | 1명/라운드 | 프리패스 발생 |

#### 3-2-3. 프리패스 공정성 규칙

```
┌─────────────────────────────────────────────────────────────────┐
│                    프리패스 공정성 시스템                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  [규칙 1] 프리패스 이력 추적                                      │
│     → 각 멤버의 프리패스 횟수를 누적 기록                          │
│     → freePassCount 필드로 관리                                  │
│                                                                 │
│  [규칙 2] 프리패스 대상자 선정 (홀수 인원 시)                       │
│     → 프리패스 횟수가 가장 적은 멤버 중 랜덤 선정                   │
│     → 동률이면 가장 최근에 당번을 수행한 멤버 우선 프리패스          │
│                                                                 │
│  [규칙 3] 프리패스 → 다음 라운드 우선 배정                         │
│     → 이번 라운드에서 프리패스를 받은 멤버는                        │
│       다음 라운드의 Week 1에 반드시 포함                           │
│     → 연속 2라운드 프리패스 금지                                   │
│                                                                 │
│  [규칙 4] 프리패스 카운트 리셋                                     │
│     → 모든 멤버가 최소 1회 프리패스를 받으면 카운트 리셋            │
│     → 15명 기준 15라운드(약 105주 ≈ 2년) 후 리셋                  │
│                                                                 │
│  [규칙 5] 짝수 인원 전환 시                                       │
│     → 인원이 짝수로 바뀌면 프리패스 없이 전원 배정                  │
│     → 기존 프리패스 카운트는 보존 (다시 홀수가 될 수 있으므로)       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3-4. F3: 랜덤 배정 엔진

#### 3-4-1. 배정 규칙

```
[규칙 1] 한 라운드 내에서 배정 멤버는 정확히 1회만 배정
[규칙 2] 항상 2인 1조 — 예외 없음
[규칙 3] 홀수 인원 시 1명은 프리패스 (프리패스 공정성 규칙 적용)
[규칙 4] 직전 라운드와 동일한 2인 조합 금지 (hard rule)
[규칙 5] 최근 3라운드 내 동일 조합 회피 (soft rule)
[규칙 6] 프리패스 받은 멤버는 다음 라운드 Week 1 우선 배정
[규칙 7] 주차 순서도 매 라운드 랜덤
```

#### 3-4-2. 알고리즘 상세

```
function generateRound(activeMembers, pairingHistory, freePassHistory):

    N = activeMembers.length

    [Step 1] 프리패스 대상 결정 (N이 홀수인 경우만)
             → freePassCount가 가장 낮은 멤버 중 랜덤 선정
             → 직전 라운드 프리패스 멤버는 제외 (연속 방지)
             → 선정된 멤버의 freePassCount +1

    [Step 2] 배정 멤버 리스트 구성
             → 전체 - 프리패스 멤버 = 배정 대상 (짝수)
             → 직전 프리패스 멤버가 있으면 Week 1 슬롯에 고정

    [Step 3] 배정 대상 셔플 (Fisher-Yates)
             → Week 1 고정 멤버 1명 + 셔플된 나머지에서 1명 = Week 1 조
             → 나머지를 순서대로 2명씩 페어링

    [Step 4] 조합 검증
             → 직전 라운드 조합과 겹치면 → Step 3으로 재셔플
             → 최대 100회 시도 후 실패 시 soft rule(규칙 5) 완화

    [Step 5] 조를 주차에 랜덤 배치 (Week 1은 고정)

    [Step 6] 결과 저장 + 이력 업데이트

    return { schedule, freePassMember }
```

#### 3-4-3. 중복 방지 수명

| 멤버 수 | 가능한 2인 조합 (ₙC₂) | 라운드당 소모 | 완전 비중복 가능              |
| ------- | --------------------- | ------------- | ----------------------------- |
| 15명    | 105                   | 7             | ~15 라운드 (약 105주 ≈ 2년)   |

#### 3-5-1. 변경 유형

```
┌──────────────────────────────────────────────────────────────┐
│                    대체자 변경 유형                             │
├──────────────┬───────────────────────────────────────────────┤
│              │                                               │
│  1:1 교환     │  A와 B가 서로의 주차를 교환                     │
│  (가장 흔함)  │  예: 지니(W2) ↔ 이바(W5)                       │
│              │  → 두 사람 모두 동의 필요                        │
│              │                                               │
├──────────────┼───────────────────────────────────────────────┤
│              │                                               │
│  프리패스     │  해당 라운드 프리패스 멤버가 자원하여 대신 수행   │
│  멤버 투입    │  → 프리패스 멤버가 대리 수행                     │
│  (홀수 시)    │  → 원래 당번 멤버가 프리패스를 이어받음           │
│              │  → 프리패스 카운트는 원래 당번에게 +1             │
│              │                                               │
├──────────────┼───────────────────────────────────────────────┤
│              │                                               │
│  대리 투입    │  A가 못하는 주차에 B가 대신 수행                 │
│              │  → B는 해당 라운드에서 2회 수행                   │
│              │  → B에게 다음 라운드 프리패스 우선권 부여          │
│              │                                               │
├──────────────┼───────────────────────────────────────────────┤
│              │                                               │
│  주차 스킵    │  공휴일/전사 이벤트 등으로 해당 주 전체 스킵      │
│  (관리자)     │  → 해당 주 당번은 다음 주로 이월                │
│              │  → 전체 라운드 일정이 1주씩 밀림                  │
│              │                                               │
└──────────────┴───────────────────────────────────────────────┘
```

#### 4-4-3. 간격 시스템 (Spacing)

```
  4px 기반 스케일 (Tailwind 호환)
  ──────────────────────────────────────────────

  --space-1     4px      ← 인라인 요소 간격
  --space-2     8px      ← 아이콘과 텍스트 사이
  --space-3    12px      ← 카드 내부 요소 간격
  --space-4    16px      ← 카드 패딩, 리스트 항목 간격
  --space-5    20px      ← 섹션 내부 그룹 간격
  --space-6    24px      ← 카드 간 간격
  --space-8    32px      ← 섹션 간 간격
  --space-12   48px      ← 페이지 섹션 구분
  --space-16   64px      ← 페이지 상하 여백

  핵심 원칙: "여백이 곧 디자인이다"
  → 요소 사이 최소 16px, 섹션 사이 최소 32px 유지
  → 모바일에서도 여백 축소하지 않음 (정보 밀도 < 가독성)
```

## 9. 사무실 이용 가이드 (보드 내 상시 게시)

보드의 "이용가이드" 탭에 상시 게시. 청소 당번과 별개로 전 구성원이 지켜야 할 규칙.

### ♻️ 분리수거

- 개인 쓰레기통 → 일반 쓰레기만
- 캔/병/플라스틱 → 화장실 앞 분리수거함
- 음식물 → 외부 쓰레기통 (사무실 내 금지)
- 목요일 퇴근 전 개인 책상 위 음식물류 정리 필수

### 🧽 회의실 사용 후

- 개인 물품 및 쓰레기 반드시 수거
- 테이블 정리, 의자 밀기는 사용자 직접

### ☕ 공용 공간

- 커피/라면 머신 사용 후 정리 (이물질 닦기, 캡슐 버리기)
- 사용한 컵/텀블러는 개인이 세척
- 공용 물건(청소기, 리모컨 등) 사용 후 제자리

### 🚨 기타

- 바닥 물/음료 흘림 → 즉시 닦기
- 청소 도구는 정해진 위치에 보관, 누구나 사용 가능
- 눈에 보이는 오염/쓰레기는 발견자가 처리하는 문화 만들기
